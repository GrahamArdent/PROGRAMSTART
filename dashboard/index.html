Create Planning Workspace

This screen creates a new standalone planning repo from the PROGRAMSTART template. It does not define the product idea yet.

What This Screen Does

It creates a fresh planning workspace, stamps in the repo name and planning rigor, copies the core PROGRAMBUILD files, and initializes a new git repository outside the template repo.

What Happens Next

1. Preview the workspace to confirm the path and files look correct.
2. Create the workspace as a new standalone repo.
3. Open that new folder in VS Code.
4. Define what you are building in the kickoff packet, canonical docs, and variant playbook.
5. Run the first planning step from the new repo.
Preview first. The create button stays disabled until the preview passes.
